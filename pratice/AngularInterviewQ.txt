Angular Theory Interview Questions (No Solutions)


#1. Core Concepts
1. What is Angular and how does it differ from AngularJS?
-> Angular is typescript based, AngularJs is Javascript based
    Angular has component architecture, AngularJs has MVC basarchitectureed
    Angular is faster, has AOT also ivy rendering engine, AngularJs JIT compilation
    Angular is Signals, zoneless feature, control flow.

2. What are the 9 building blocks of Angular?
-> Modules, Components, Templates, Directives, Services, Dependency Injection, Router, HTTP Client, and State Management.

3. Explain template vs component vs service
Template = HTML view (what users see).
Component = TypeScript class + logic (what makes template dynamic).
Service = Business logic/data layer (shared across components).

4. Difference between Declaration vs Provider in NgModule
-> Declarations register view classes (components, directives, pipes) local to the module.
   Providers register injectable services for Dependency Injection (DI).

5. What is dependency injection (DI)?
-> Dependency Injection is a design pattern where dependencies are provided to the class rather than creating its own.
So anything can be used as dependency who has @injectable decorator. Services, Gaurds, Interceptors, resolvers.
Enables loose coupling, reusability.


6. What is AOT compilation vs JIT? before angular 9 its JIT
-> AOT - complies at build time component into javascript bundle files and runs it on Browser
   JIT - complies at run time, converts into bundlefiles and downloads it on browser with compiler.

Angular CLI purpose and key commands (ng generate, ng serve)

#2. Components & Lifecycle
7. constructor() vs ngOnInit() - when to use each?
-> contructor is used to inject service into components. ngOnInit is used to initial the component and its variables.

8. List all 8 lifecycle hooks in execution order
-> ngOnChanges() - Input property changes
   ngOnInit() - Component/data initialization
   ngDoCheck() - Custom change detection
   ngAfterContentInit() - Content projection initialized
   ngAfterContentChecked() - Content projection checked
   ngAfterViewInit() - View and child views initialized
   ngAfterViewChecked() - View and child views checked
   ngOnDestroy() - Cleanup before component removal 


9. What triggers ngOnChanges()?
-> ngOnChanges() triggers when Angular detects changes to @Input() properties bound via template property 
binding (e.g., [inputName]="parentValue"). It receives a SimpleChanges object tracking previous and 
current values

10. When does ngOnDestroy() execute?
-> ngOnDestroy() executes once just before Angular destroys the component or directive. It's the final cleanup hook in the lifecycle sequence.

11. @Input() vs @Output() decorator usage
-> It implement parent-child relationship. @Input() passes data from parent to child components. @Output() emits events from child to parent via EventEmitter

#5. Pipes
16. Pure vs impure pipes - performance impact
->  
| Type                 | When Executes                            | Performance Cost                        |
| -------------------- | ---------------------------------------- | --------------------------------------- |
| Pure (default)       | Input primitive/object reference changes | ✅ Fast - skips unchanged data           |
| Impure (pure: false) | Every change detection cycle             | ❌ Slow - mouse moves, timers trigger it |

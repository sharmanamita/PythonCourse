14. Difference between append() and extend().
-> append() adds elements as one, extend() expand the elements adds it one by one like flattening.
my_list = [1, 2, 3]

# append() - adds entire list as ONE element
my_list.append([4, 5])
print(my_list)  # [1, 2, 3, [4, 5]]

# extend() - adds EACH element separately
my_list.extend([4, 5])
print(my_list)  # [1, 2, 3, 4, 5]

# Strings work too!
my_list = ['hello']
my_list.append('world')      # ['hello', 'world']
my_list.extend('world')      # ['hello', 'w', 'o', 'r', 'l', 'd']

17. What is exception handling? try-except-finally.
-> "Exception handling in Python uses try-except-finally blocks to catch and handle runtime errors.

try: Contains risky code
except: Handles specific errors  
else: Runs only if no exception occurs
finally: Always executes (cleanup)

try:
    # Risky code that might fail
    risky_operation()
except ValueError:
    # Handle specific error
    print("Invalid value!")
except Exception as e:
    # Catch-all for other errors
    print(f"Error: {e}")
else:
    # Only runs if NO exception
    print("Success!")
finally:
    # ALWAYS executes (cleanup)
    print("Cleanup complete")


Key principle: Fail gracefully, never use bare except."

19. What does len() do? Works on what types?
->  len() is a built-in function that returns an integer representing the number of items in sequences or collections.

Syntax: len(object)

Works on: strings, lists, tuples, dicts, sets
Doesn't work on: integers, custom objects (unless __len__ implemented)

| Type   | Example      | len() Result        |
| ------ | ------------ | ------------------- |
| String | len("hello") | 5                   |
| List   | len([1,2,3]) | 3                   |
| Tuple  | len((1,2))   | 2                   |
| Dict   | len({'a':1}) | 1 (key-value pairs) |
| Set    | len({1,2})   | 2                   |

20. What is PEP 8? Why follow it?
-> PEP 8 = Python Enhancement Proposal 8 - coding conventions document.

**Key Rules:**
- 4 spaces indentation (no tabs)
- Max 79 chars per line
- snake_case for functions/variables
- CamelCase for classes  
- Single quotes preferred
- Blank lines: 2 between functions, 1 between methods

**Why follow?** Team consistency, readability, easier maintenance.

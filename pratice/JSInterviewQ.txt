6. what is call/apply/bind purpose?
-> call(), apply(), and bind() control the this context in JavaScript functions.
    These methods are essential for function borrowing, event handling, and argument passing
    They solve issues like losing this context in callbacks or borrowing methods between objects.

    call - Invoke a function immediately with individual arguments and a specific this.
    apply - Invoke a function immediately with array of arguments and a specific this.
    bind - Returns a new function with fixed this and optional pre-set arguments, without immediate execution.

12. Arrow functions vs regular functions (this binding)
-> Arrows are perfect for callbacks but never use them as constructors, object methods, or
   when you need dynamic this or arguments. They have lexical this—fixed at definition time.

   Lexical this means arrow functions inherit this from their surrounding (enclosing) scope where defined, 
   not from how they're called

    | Aspect          | Regular Function    | Arrow Function                          |
    | --------------- | ------------------- | --------------------------------------- |
    | this Source     | Dynamic (call site) | Lexical (enclosing scope) freecodecamp ​ |
    | call/apply/bind | Can override this   | Ignores—uses lexical this               |
    | Object Methods  | this = object       | this = outer scope                      |
    | Callbacks       | Needs .bind(this)   | Works automatically                     |

14. Spread operator vs Rest parameters
->  Spread operator expands iterables into individual elements; 
    Rest parameters collect remaining arguments into an array. 
    Both use ... syntax but opposite purposes—spread "unpacks," rest "packs."

    // Array copy + merge // Spread
    const a = [1, 2]; const b = [...a, 3, 4];  // [1,2,3,4]
    // Object merge
    const obj1 = {a: 1}; const obj2 = {...obj1, b: 2};  // {a:1, b:2}

    // Variable args // rest
    function sum(...numbers) {
        return numbers.reduce((a, b) => a + b, 0);
    }
    sum(1, 2, 3, 4);  // 10

    // Destructuring rest
    const [first, ...rest] = [1, 2, 3, 4];  // first=1, rest=[2,3,4]
    const {name, ...profile} = {name: 'Nami', age: 30};  // profile={age:30}

19. Promise vs async/await vs callbacks
->  
#Promise: is a javascript object which represents the completion or failure of async operations.
Its acts like a placeholder to a value which may not be ready yet. It has two main elements state and result.
state has different status like pending, fullfilled or failed. 
It resolves the issue of callback hell with multiple chaining of .then .catch()

Promise.all([promise1, promise2])        // All must succeed
Promise.allSettled([promise1, promise2]) // All must complete  
Promise.race([promise1, promise2])       // First to settle
Promise.any([promise1, promise2])        // First success

function httpGetPromise(url) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    
    xhr.onreadystatechange = function() {
      if (xhr.readyState === 4) {  // DONE
        if (xhr.status === 200) {
          resolve(JSON.parse(xhr.responseText));
        } else {
          reject(new Error(`HTTP ${xhr.status}: ${xhr.statusText}`));
        }
      }
    };
    
    xhr.onerror = function() {
      reject(new Error('Network error'));
    };
    
    xhr.open('GET', url);
    xhr.send();
  });
}

// REAL API CALL
httpGetPromise('https://jsonplaceholder.typicode.com/posts?userId=1')
  .then(posts => {
    console.log(`✅ Got ${posts.length} posts`);
    console.table(posts.slice(0, 3));  // First 3 posts
  })
  .catch(error => {
    console.error('❌ Error:', error.message);
  });

#Aysnc/await: lets you write aysnc promise-based code which look and read like a synchronous JS.
// Promise chains (harder to read)
fetch(url)
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async/await (cleaner)
async function getData() {
  try {
    const res = await fetch(url);
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

| Pattern     | When to use                                  | Execution style           |
| ----------- | -------------------------------------------- | ------------------------- |
| async/await | Sequential operations, readable control flow | One-by-one (await pauses) |
| Promise.all | Multiple independent parallel operations     | All at once (concurrent)  |
